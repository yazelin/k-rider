# K-Rider Case Study：一個遊戲怎麼用「甲方思維」做出來

這是 [AI 互動行銷頁實作課](https://yazelin.github.io/ai-marketing-pages-course/) 各進階模組堆到產品級的樣子。課程教你零 build、一頁活動頁；K-Rider 把同一套技能（視覺素材、特效、部署排程、AI Worker、名單漏斗）推到一個有真實股價資料的網頁遊戲。程式碼是耗材，規格與驗收才是帶得走的——這份 case study 不貼程式碼，只拆「規格怎麼開、發包怎麼下、驗收怎麼收」。

成品在這裡玩：<https://yazelin.github.io/k-rider/>

---

## 一、一頁規格書

開工前先把甲方規格寫清楚。一頁，四格，講完就能發包。

- **受眾**：想培養「盤感」、對股價波動有體感，但不想真金白銀進場賠錢的人。把抽象的紅綠 K 線變成可以親手騎過去的地形。
- **核心玩法價值**：真實歷史股價 → 賽道地形。台股紅漲綠跌、美股反過來；騎機車衝過大波動日，沿途撞上 AI 生成的歷史事件路牌，摔車後被 AI 嘴一句。把「看不懂的線」變成「騎得過去的坡」。
- **區塊**：選關（首頁每日挑戰卡 + 精選賽道 + 任意 ticker 搜尋）→ 騎乘（Matter.js 物理 + HUD + 小地圖）→ 結算（分數、特技字典、戰績梗圖）→ 分享（梗圖貼 LINE / Threads / X / FB / Reddit）+ 留資（結算/聲明頁底部換每日挑戰提醒）。
- **驗收**：地形保證可玩（坡度上限壓回 ~50°）、機車是單一剛體（輪距幾何上不可能變形）、手感數值以 headless 模擬實測定案、AI 賽評共享快取只開放精選股以防 prompt injection 投毒。

範圍刻意切掉的（v2 backlog）：重度防作弊（重播驗證、伺服器端物理）、TWSE 備援資料源、玩家帳號系統、自訂車輛。先做出能玩的天花板，不做防禦性的完美。

## 二、prompt 鏈：怎麼「下發包」給 AI

不是一句話叫 AI「做個遊戲」，而是把規格拆成一段一段可驗收的發包。每一段都附一句關鍵 prompt，體現「給規格、收成品、驗收、再迭代」。

1. **設計 mock（先有畫面才好對齊）**
   用 Codex `$imagegen` 先把 HUD 與結算卡畫成圖（`docs/design/mock-game-screen.png`、`mock-settle-card.png`），照稿落 CSS，而不是邊寫邊猜版面。
   > 發包：「畫一張暗色 neon 風的機車越野遊戲 HUD：左上累計分數、上方計時與氮氣條、右上全賽道縮圖小地圖；另一張結算卡：完賽/摔車標題、里程滯空空翻耗時四格數據、虛擬本金 10 萬的損益。自有配色，不抄任何現有遊戲素材。」

2. **物理調參迭代（手感用實測定案，不靠手感喊）**
   機車物理不靠「感覺差不多」，而是寫 headless 模擬，把重力、跳力、坡度目標當參數實測迭代。
   > 發包：「機車是單一剛體 compound（車架/騎士/兩輪都是 parts），驅動用沿坡面純力模型 + 角速度導引姿態控制，接地用法向距離幾何判定。寫測試證明：落平地 3 秒內穩定不抖、35° 連續上坡油門爬得上去、亂力轟炸 10 秒輪距恆定。」

3. **AI 路牌生成（離線增量，缺了不擋）**
   掃 12 檔日線找單日漲跌 |≥5%|（ETF/SPY |≥3%|）的日期，對「尚無路牌」的日期呼叫 LLM 生一句雙語事件摘要，存進 `public/data/events/`。已生成過的不重打。
   > 發包：「給你某檔某天的漲跌幅，回一句雙語（zh ≤40 字 / en ≤25 words）說明那天市場最可能的背景。生過的日期跳過不重打，LLM 掛掉就讓那天沒路牌，不要擋住整條資料 workflow。」

4. **漏斗（免費價值先給，再換 email）**
   遊戲免費玩，玩到結算頁/聲明頁底部才出現留資表單，留 email 換「每日挑戰提醒」，成功當場兌現一條精選賽道連結。
   > 發包：「結算頁底掛一個零依賴 email 表單，藏一個 honeypot 欄位擋機器人。送 `POST /signup`：KV 近似限流、D1 `UNIQUE(email)` 去重，回一條精選賽道連結當場兌現；重複留資也照樣再給連結。名單用一個 `noindex` 的 `admin.html` 憑 Bearer token 看。」

## 三、驗收清單

規格寫了不算數，要有可重複跑的驗收。K-Rider 的驗收是真的 vitest 測試（`npm test`，78 個），不是嘴上的「測過了」。挑幾條代表性的：

- **機車落地穩定不抖、不穿地**：平地放 3 秒 settle，再 1 秒，車身 y 位移 < 2px、角度 < 0.3 rad、人在地面上方。下坡地形滑 10 秒不沉到地面下。（`tests/physics.test.js`）
- **爬坡力回歸測試**：35° 連續上坡，全程油門 15 秒（同 `run.js` 的沿坡推力 + 姿態控制器），驗證車真的爬過坡段中後段——改物理參數不准把這條弄壞。
- **輪距不變量**：亂力轟炸 10 秒（大跳、亂轉、亂推），每一幀斷言前後輪距與初始值差 < 0.01px。因為整台車是單一剛體 compound，輪胎相對車身在幾何上不可能變形——這條不是「應該不會變」，是「結構上不可能變」，測試把它釘死。
- **手感數值實測定案**：重力、跳力、坡度目標這些參數不靠手感喊，用 headless `Matter.Engine.update` 跑出來實測；spec 也明列遊戲手感另以「canvas 非透明像素數 > 閾值」做頁面載入煙霧測試。
- **計分上限重算 + 暱稱清洗**：`POST /score` 用同一份 `scoring.js` 以該賽道資料點數推理論最大值重算，超過上限直接擋；暱稱去 HTML、≤16 字；每 IP 每日限流。前後端、Worker 共用同一份純函式，不會兩邊算不一樣。
- **roast 共享快取防 prompt injection**：賽後 AI 賽評（Groq）的 KV 快取**只開放給精選股**。任意 symbol 不進共享快取，避免有人用自訂 ticker 帶 prompt injection 去投毒其他玩家會讀到的快取行；user 給的 symbol / period 字串也先截斷長度再進 prompt。（`worker/src/roast.js`）
- **資料管線冪等**：GitHub Actions 多 cron 抓 Yahoo，輸出內容和現有檔一樣（忽略 `updated` 欄）就不寫入、無 diff 就不 commit；驗證不過保留舊檔。被擋一次站不會壞。

## 四、模組對應表

把課程的進階模組一格一格對到 K-Rider 的實作。同一套技能，課程教在一頁活動頁上，K-Rider 推到一個有真實資料的遊戲。

| 課程模組 | K-Rider 對應 |
|---|---|
| 模組 2 視覺素材 | Codex `$imagegen` 出 HUD/結算卡 mock → 照稿落 CSS |
| 模組 3 特效 | 零依賴 canvas（星空/光柱/金塵）+ WebAudio 全合成音效 |
| 模組 5/7 部署排程 | GitHub Actions 多 cron 抓 Yahoo（冪等）+ Pages |
| 模組 6 AI Worker | roast 吐槽（共享快取防 prompt injection）+ AI 事件路牌 |
| 模組 9 名單漏斗 | `/signup` + D1 + `admin.html`（本次新增） |
